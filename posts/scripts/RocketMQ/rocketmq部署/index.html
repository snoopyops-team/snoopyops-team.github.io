墨西哥生产环境Rocketmq部署

参考安装方式: https://github.com/apache/rocketmq/blob/master/docs/cn/operation.md

与研发确认采用2m-2s-async模式部署。

采用4台服务器，master上部署namesrv。

安装Java 1.8。

#####启动服务:

如果服务器是8g内存，需先修改java启动配置

JAVA_OPT="${JAVA_OPT} -server -Xms4g -Xmx4g -Xmn2g -XX:MetaspaceSize=128m -XX:MaxMetaspaceSize=320m"

修改为

JAVA_OPT="${JAVA_OPT} -server -Xms3g -Xmx3g -Xmn2g -XX:MetaspaceSize=128m -XX:MaxMetaspaceSize=320m"

JAVA_OPT="${JAVA_OPT} -server -Xms8g -Xmx8g -Xmn4g"

修改为

JAVA_OPT="${JAVA_OPT} -server -Xms2g -Xmx2g -Xmn1g"
  • 启动namesrv: 在rocketmq-01/02上执行:
cd /work/rocketmq && nohup sh bin/mqnamesrv &
  • 启动broker： 在rocketmq-01上执行:
nohup sh bin/mqbroker -c /work/rocketmq/conf/2m-2s-async/broker-a.properties >/dev/null 2>&1 &

在rocketmq-02上执行:

nohup sh bin/mqbroker -c /work/rocketmq/conf/2m-2s-async/broker-b.properties >/dev/null 2>&1 &

03和04是纯粹的broker，所以可调整为4g（如果服务器内存为8g）

JAVA_OPT="${JAVA_OPT} -server -Xms8g -Xmx8g -Xmn4g"

修改为

JAVA_OPT="${JAVA_OPT} -server -Xms4g -Xmx4g -Xmn1g"

在rocketmq-03上执行:

nohup sh bin/mqbroker -c /work/rocketmq/conf/2m-2s-async/broker-a-s.properties >/dev/null 2>&1 &

在rocketmq-04上执行:

nohup sh bin/mqbroker -c /work/rocketmq/conf/2m-2s-async/broker-b-s.properties >/dev/null 2>&1 &

如果服务起不来，看机器配置调整/work/rocketmq/bin下runserver.sh和runbroker.sh的java启动参数。 启动顺序：先启动NameServer，再启动BrokerServer（启动BrokerServer之前先关闭防火墙)。 停止顺序：先停止BrokerServer，再停止NameServer。 停止broker命令: cd /work/rocketmq/bin && sh mqshutdown broker 停止namesrv命令: cd /work/rocketmq/bin && sh mqshutdown namesrv

启动服务

cd target/ && nohup java -jar rocketmq-console-ng-2.0.0.jar --server.port=8080 --rocketmq.config.namesrvAddr=10.122.0.98:9876\;10.122.0.33:9876 &
这里需要注意三个参数：
rocketmq-console-ng-2.0.0.jar 请注意查看实际的版本，
--server.port=8080   指定console访问端口，默认的就是8080，也可以指定为其他端口
--rocketmq.config.namesrvAddr   指定nameserver的地址

#####RocketMQ常用命令:

需要切换到bin目录下，即：
cd /work/rocketmq/bin
 
获取所有可用命令：
sh mqadmin
 
查看帮助：
sh mqadmin <command> -h
查询Producer的网络连接情况：
sh mqadmin producerConnection -n localhost:9876 -g <producer-group> -t <producer-topic>
查询Consumer的网络连接情况：
sh mqadmin consumerConnection -n localhost:9876 -g <consumer-group>
查询Consumer的消费状态：
sh mqadmin consumerProgress -n localhost:9876 -g <consumer-group>
 
查询消息是否发送成功
获取指定Topic：
sh mqadmin topicList -n localhost:9876 | grep <topicName>
查看Topic状态：
sh mqadmin topicStatus -n localhost:9876 -t <topicName>
根据offset获取消息：
sh sh mqadmin queryMsgByOffset -n localhost:9876 -b <broker-name> -i <queueId> -o <offset> -t <topicName>
根据offsetMsgId查询消息：
sh sh mqadmin queryMsgById -n localhost:9876 -i <offsetMsgId>
 
查询消息是否被消费成功
查询消息详情：
sh mqadmin queryMsgById -i {MsgId} -n {NameServerAddr}
查看Consumer Group订阅了哪些TOPIC：
sh mqadmin consumerProgress -g <ConsumerGroup> -n <NameServerAddr>
 
查询TOPIC被哪些Consumer Group订阅了
没有查询特定TOPIC订阅情况，只能查询所有后再过滤：
sh mqadmin statsAll -n <NameServerAddr> | grep <TOPIC>
返回结果：#Topic #Consumer Group #InTPS #OutTPS #InMsg24Hour #OutMsg24Hour
 
关闭nameserver和所有的broker:
sh mqshutdown namesrv
sh mqshutdown broker
 
查看所有消费组group:
sh mqadmin consumerProgress -n 192.168.23.159:9876
查看指定消费组（kevinGroupConsumer）下的所有topic数据堆积情况：
sh mqadmin consumerProgress -n 192.168.23.159:9876 -g kevinGroupConsumer
查看所有topic :
sh mqadmin topicList -n 192.168.23.159:9876
查看topic信息列表详情统计
sh mqadmin topicstatus -n 192.168.23.159:9876 -t myTopicTest1
新增topic
sh mqadmin updateTopic –n 10.45.47.168 –c DefaultCluster –t ZTEExample
删除topic
sh mqadmin deleteTopic –n 10.45.47.168:9876 –c DefaultCluster –t ZTEExample
